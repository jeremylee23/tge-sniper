# 安全指南

## 私鑰保護

### 加密機制

本工具使用以下方式保護私鑰：

- **AES-256-GCM** - 軍用級加密演算法
- **scrypt** - 密鑰派生函數，防止暴力破解
- **主密碼** - 所有私鑰使用單一主密碼加密

### 最佳實踐

1. **設定強密碼**
   - 至少 12 個字元
   - 包含大小寫、數字、符號
   - 不要使用常見密碼

2. **安全儲存**
   - `data/` 目錄包含加密的私鑰
   - 定期備份此目錄
   - 不要上傳到版本控制 (已在 .gitignore)

3. **操作安全**
   - 在隔離環境中運行
   - 不要在共用電腦上使用
   - 使用完畢執行 `wallet lock`

## 風險警告

### 資金風險

⚠️ **重要提醒**

- 搶購可能失敗，Gas 費用無法退回
- 高 Gas 競價可能導致高額費用
- 測試網測試不保證主網成功

### 緩解措施

1. **先用測試網**
   - 使用 Sepolia (ETH) 或 Devnet (SOL)
   - 確認流程正確後再上主網

2. **小額測試**
   - 先用少量資金測試
   - 確認成功後再增加投入

3. **分散錢包**
   - 使用多個錢包分散風險
   - 不要將所有資金集中

## 環境安全

### 建議措施

```bash
# 1. 使用專用 Node.js 環境
nvm use 20

# 2. 定期更新依賴
npm update

# 3. 檢查依賴安全性
npm audit
```

### 不要做

❌ 將 `.env` 上傳到 GitHub  
❌ 分享主密碼  
❌ 在公共網路執行搶購  
❌ 使用來路不明的 RPC

## 應急處理

### 密碼遺失

```bash
# 無法找回密碼
# 唯一選項是刪除資料重新開始
rm -rf data/
npm run cli wallet init
```

### 私鑰匯出

如需備份私鑰：

```bash
npm run cli wallet export --address 0x...
# ⚠️ 私鑰會顯示在終端機，請小心處理
```

### 緊急停止

執行中的搶購任務：

```bash
# 按 Ctrl+C 停止
# 已廣播的交易無法取消
```
